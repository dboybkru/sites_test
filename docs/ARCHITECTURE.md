# Архитектура лаборатории

## Идея
Монорепо-«лаборатория» под много дизайн-экспериментов. Три слоя:

1. **Рыбы** (`templates/`) — стартовые шаблоны, ОТКУДА копируется новый сайт.
2. **Фишки** (`packages/`) — переиспользуемые токены, анимации, компоненты.
3. **Сайты** (`sites/`) — самодостаточные эксперименты. Каждый можно собрать и продать отдельно.

Витрина (`showcase/`) собирает все сайты в одну галерею.

## Почему не взяли `di-sukharev/vibe` целиком
`vibe` — тяжёлый full-stack монорепо под ОДИН продукт (Bun + Hono + Prisma + Postgres,
React webapp, Astro website, Expo mobile, деплой на DigitalOcean). Из него взяли идеи:
- Astro для статичных/SEO-страниц и витрины;
- дисциплину монорепо (workspaces, общий `packages/`, `docs/`).

Выбросили бэкенд/мобайл/Postgres — для дизайн-экспериментов не нужны.
Если конкретному сайту понадобится бэкенд — добавим рыбу `templates/vibe-fullstack/`
(клон vibe) и используем её точечно для этого сайта.

## Поток создания сайта
```
npm run new -- 02-glass-hero "Glass Hero"   # копия рыбы в sites/02-glass-hero
npm install                                  # подтянуть workspace-зависимости
npm run dev --workspace @site/02-glass-hero  # разработка
```
Заполни `sites/02-glass-hero/site.meta.json` — карточка появится в витрине.

## Сборка и деплой
- `npm run build:all` собирает каждый сайт под `/s/<slug>/` и кладёт в `showcase/public/s/`,
  затем собирает витрину → единый `showcase/dist`.
- Vercel настроен (`vercel.json`) на `build:all` + output `showcase/dist`.
- Подпуть `/s/<slug>/` обеспечивается переменной `SITE_BASE` (см. `astro.config.mjs` рыбы).

## Правило переиспользования
Скопировал паттерн дважды — на третий раз выноси в `packages/ui`.
Так фишки растут органически, а сайты остаются автономными (продаваемыми).
