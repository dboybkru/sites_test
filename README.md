# test_site_design — дизайн-лаборатория

Монорепо для экспериментов с дизайном сайтов: разные подходы, скилы и инструменты.
Каждый эксперимент — самодостаточная папка в `sites/`, которую можно показать, переиспользовать или **продать целиком**.

## Структура

```
showcase/        Витрина-галерея (Astro) — собирает все сайты в одну страницу. Деплоится на Vercel.
packages/        Общие «фишки», переиспользуемые между сайтами:
  tokens/          @lab/tokens — цвета, типографика, spacing (CSS-переменные)
  motion/          @lab/motion — анимации/keyframes-пресеты
  ui/              @lab/ui     — общие компоненты
templates/       «Рыбы» — стартовые шаблоны, ОТКУДА копируется новый сайт (не workspace)
  astro-static/    Лёгкая статичная рыба на Astro
sites/           Эксперименты. Каждый автономен (свой build), фишки копируются/подключаются
  01-hello-motion/
scripts/         Утилиты монорепо (new-site, build-all)
docs/            Архитектура и заметки
```

## Команды

```bash
npm install                 # поставить всё (workspaces)
npm run dev                 # запустить витрину локально
npm run new -- <slug>       # создать новый сайт из templates/astro-static
npm run build:all           # собрать все сайты + витрину в showcase/dist (то, что уедет на Vercel)
```

## Как это закрывает цели

- **Видеть самому** — `npm run build:all` собирает галерею со всеми сайтами под `/s/<slug>/`.
- **Показать другим** — деплой `showcase/dist` на Vercel → одна публичная ссылка, превью на каждый коммит.
- **Продать** — каждый `sites/*` самодостаточен: отдаёшь одну папку.

## Деплой (Vercel)

Проект уже сконфигурирован (`vercel.json`): build = `npm run build:all`, output = `showcase/dist`.

```bash
vercel        # превью-деплой
vercel --prod # прод
```
